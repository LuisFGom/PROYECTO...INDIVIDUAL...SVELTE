                                    1. MÓDULO DE CLIENTES
═══════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════

VENTANA: Gestión de Clientes (clientes.js)
DESCRIPCIÓN: Listado, creación, edición y eliminación de clientes con búsqueda avanzada

├─ BÚSQUEDA (Campo de búsqueda en el header)
│  ├─ Tipo: TEXT - Campo libre
│  ├─ Funcionalidad: Búsqueda en tiempo real
│  ├─ Busca por: 
│  │  ├─ Nombre del cliente
│  │  ├─ Documento (con o sin formato)
│  │  ├─ Teléfono (sin formato)
│  │  ├─ Correo/Email
│  ├─ Validación: Ninguna (aceptar todos caracteres)
│  ├─ Limitación: Ninguna
│  └─ Comportamiento: Búsqueda inmediata mientras se escribe

├─ CREAR/EDITAR CLIENTE - CAMPO: NOMBRE
│  ├─ Requerido: SÍ (obligatorio)
│  ├─ Tipo: TEXT
│  ├─ Validación: Solo letras (sin números ni caracteres especiales)
│  ├─ Limitación: No tiene límite en el código, pero puede haber límite en BD
│  ├─ Regex: Acepta letras, espacios y acentos (á, é, í, ó, ú, ñ)
│  ├─ Placeholder: "Nombre"
│  └─ Comportamiento: Si se intenta ingresar números o caracteres especiales, se remueven automáticamente

├─ CREAR/EDITAR CLIENTE - CAMPO: APELLIDO
│  ├─ Requerido: SÍ (obligatorio)
│  ├─ Tipo: TEXT
│  ├─ Validación: Solo letras (sin números ni caracteres especiales)
│  ├─ Limitación: No tiene límite explícito en el código
│  ├─ Regex: Acepta letras, espacios y acentos (á, é, í, ó, ú, ñ)
│  ├─ Placeholder: "Apellido"
│  └─ Comportamiento: Si se intenta ingresar números o caracteres especiales, se remueven automáticamente

├─ CREAR/EDITAR CLIENTE - CAMPO: CÉDULA/DOCUMENTO (Solo en CREACIÓN)
│  ├─ Requerido: SÍ (solo en creación, en edición ESTÁ DESHABILITADO)
│  ├─ Tipo: TEXT con máscara
│  ├─ Formato: XX-XXX-XXX-X (10 dígitos totales)
│  ├─ Limitación: Máximo 13 caracteres (incluyendo guiones)
│  ├─ Validación:
│  │  ├─ Solo acepta números (los guiones se agregan automáticamente)
│  │  ├─ Exactamente 10 dígitos numéricos
│  │  ├─ Regex: /^\d{10}$/
│  ├─ Máscara automática: 
│  │  ├─ Mientras escribes: 1 -> 1 | 12 -> 12 | 123 -> 12-3 | 1234 -> 12-345 | 12345 -> 12-345-6 | 123456 -> 12-345-67 | 1234567 -> 12-345-678 | 12345678 -> 12-345-678-9 | 123456789 -> 12-345-678-90
│  ├─ Placeholder: "17-056-789-01 (formato: XX-XXX-XXX-X)"
│  ├─ Nota: En edición está deshabilitado (gris) para evitar cambio accidental
│  └─ Validación adicional: 
│      └─ Error si: Documento vacío en creación NUEVO
│      └─ Error si: Documento no tiene exactamente 10 dígitos

├─ CREAR/EDITAR CLIENTE - CAMPO: EMAIL
│  ├─ Requerido: NO (opcional)
│  ├─ Tipo: EMAIL
│  ├─ Validación: 
│  │  ├─ Si se deja vacío: OK (no valida)
│  │  ├─ Si se ingresa algo: Debe ser email válido con punto después del @
│  │  ├─ Regex: /^[^\s@]+@[^\s@]+\.[^\s@]+$/
│  ├─ Placeholder: "correo@ejemplo.com"
│  └─ Error si: Email ingresado pero inválido (ej: "usuario@.com" sin dominio)

├─ CREAR/EDITAR CLIENTE - CAMPO: TELÉFONO
│  ├─ Requerido: NO (opcional)
│  ├─ Tipo: TEXT con máscara
│  ├─ Formato: 09-XXXX-XXXX (10 dígitos, generalmente para Ecuador)
│  ├─ Limitación: Máximo 12 caracteres (incluyendo guiones)
│  ├─ Validación:
│  │  ├─ Solo acepta números (los guiones se agregan automáticamente)
│  │  ├─ Entre 7 y 10 dígitos numéricos
│  │  ├─ Regex: /^\d{7,10}$/
│  ├─ Máscara automática:
│  │  ├─ Mientras escribes: 09 -> 09 | 098 -> 09-8 | 0987 -> 09-87 | 09876 -> 09-876 | 098765 -> 09-8765 | 0987654 -> 09-8765-4 | 09876543 -> 09-8765-43 | 098765432 -> 09-8765-432 | 0987654321 -> 09-8765-4321
│  ├─ Placeholder: "09-8765-4321 (formato: 09-XXXX-XXXX)"
│  ├─ Nota adicional: "7-10 dígitos, máscara automática 09-XXXX-XXXX"
│  └─ Error si: Teléfono ingresado pero no cumple con 7-10 dígitos

├─ CREAR/EDITAR CLIENTE - CAMPO: DIRECCIÓN
│  ├─ Requerido: NO (opcional)
│  ├─ Tipo: TEXT
│  ├─ Máximo: 100 caracteres
│  ├─ Validación: Ninguna especial (acepta todos los caracteres)
│  ├─ Placeholder: "Ej: Calle Principal 123 (máximo 100 caracteres)"
│  ├─ Nota adicional: "Máximo 100 caracteres"
│  └─ Error si: Dirección tiene más de 100 caracteres

RESTRICCIONES GENERALES DEL MÓDULO:
├─ Búsqueda: Se actualiza en tiempo real mientras escribes
├─ Tabla: Muestra máximo 10 clientes por página (paginación)
├─ Permiso: Solo ADMINS pueden crear, editar y eliminar clientes
├─ Usuarios normales: Solo pueden ver la lista de clientes
├─ Cédula: No puede ser modificada después de la creación del cliente
└─ Estado: Clientes se marcan como "Activos" o "Inactivos", no se eliminan físicamente

│  ├─ Mínimo: 0.01
│  ├─ Validación:
│  │  ├─ Solo números y UN punto decimal

├─ CREAR/EDITAR PRODUCTO - CAMPO: STOCK MÍNIMO

│  ├─ Tipo: NUMBER (solo enteros)
│  ├─ Formato: Número entero positivo
│  ├─ Mínimo: 0
│  ├─ Validación:
│  │  ├─ Solo números enteros

│  │  ├─ Regex: Solo dígitos /^[0-9]+$/

│  ├─ Valor por defecto: 0

│  ├─ Filtro en tiempo real: Cualquier carácter que no sea dígito se elimina automáticamente
│  └─ Validación adicional: Debe ser <= Stock Inicial (IMPORTANTE: Validación en guardar)

├─ CREAR/EDITAR PRODUCTO - CAMPO: IVA (%)
│  ├─ Requerido: NO (opcional, valor por defecto: 0)
│  ├─ Tipo: NUMBER decimal
│  ├─ Formato: Porcentaje (0-100)
│  ├─ Mínimo: 0
│  ├─ Máximo: 100
│  ├─ Validación:
│  │  ├─ Solo números y decimales
│  │  ├─ Entre 0 y 100
│  │  ├─ Paso: 0.01
│  ├─ Placeholder: "0"
│  ├─ Valor por defecto: 0
│  └─ Nota: Se usa en cálculos automáticos

CÁLCULOS AUTOMÁTICOS EN TIEMPO REAL:
├─ MARGEN DE GANANCIA: Precio Venta - Precio Compra
├─ MONTO IVA: (Precio Venta * IVA%) / 100
├─ PRECIO FINAL: Precio Venta + Monto IVA
├─ Formato: Moneda (ej: $10.50, $19.99)
└─ Actualización: Se recalcula automáticamente al cambiar cualquier valor

RESTRICCIONES GENERALES DEL MÓDULO:
├─ Búsqueda: Se actualiza en tiempo real mientras escribes
├─ Tabla: Muestra máximo 10 productos por página (paginación)
├─ Permiso: Solo ADMINS pueden crear, editar y eliminar productos
├─ Usuarios normales: Solo pueden ver la lista de productos
├─ Stock Mínimo: Debe ser siempre menor o igual al Stock Inicial
└─ Precios: No pueden ser negativos ni cero

├─ LÍNEA DE VENTA (Tabla de items)

│  │  ├─ Requerido: SÍ (en cada línea)
│  │  ├─ Tipo: SELECT/DROPDOWN
│  │  ├─ Validación: Producto existente con stock disponible
│  │  └─ Nota: Solo se muestran productos con stock > 0
│  │
│  ├─ CAMPO: CANTIDAD
│  │  ├─ Requerido: SÍ (en cada línea)
│  │  ├─ Tipo: NUMBER (solo enteros)
│  │  ├─ Mínimo: 1
│  │  ├─ Máximo: Stock disponible del producto
│  │  ├─ Validación:
│  │  │  ├─ Solo números enteros positivos
│  │  │  ├─ No puede exceder stock disponible
│  │  │  └─ Regex: /^\d+$/
│  │  ├─ Filtro en tiempo real: Cualquier carácter que no sea dígito se elimina
│  │  └─ Error si: Cantidad > stock disponible
│  │
│  ├─ CAMPO: PRECIO UNITARIO
│  │  ├─ Requerido: SÍ (se autocarga)
│  │  ├─ Tipo: NUMBER (solo lectura en la mayoría de casos)
│  │  ├─ Formato: Decimal con hasta 2 decimales
│  │  ├─ Validación: Se obtiene del producto
│  │  └─ Nota: Puede ser editable en algunos sistemas
│  │
│  ├─ CAMPO: SUBTOTAL
│  │  ├─ Requerido: NO (se calcula automáticamente)
│  │  ├─ Tipo: NUMBER (solo lectura)
│  │  ├─ Fórmula: Cantidad * Precio Unitario
│  │  ├─ Formato: Moneda (ej: $50.50)
│  │  └─ Nota: Se actualiza automáticamente
│  │
│  └─ ACCIÓN: ELIMINAR
│     ├─ Tipo: BOTÓN
│     ├─ Función: Elimina la línea de la venta
│     └─ Nota: Se puede agregar/eliminar líneas libremente

CÁLCULOS AUTOMÁTICOS EN TIEMPO REAL:
├─ SUBTOTAL TOTAL: Suma de todos los subtotales
├─ DESCUENTO: Si aplica (editable)
├─ SUBTOTAL DESPUÉS DESCUENTO: Subtotal Total - Descuento
├─ IVA TOTAL: (Subtotal después descuento * IVA%) / 100
├─ TOTAL FACTURA: Subtotal - Descuento + IVA
└─ Formato: Moneda

RESTRICCIONES GENERALES DEL MÓDULO:
├─ Mínimo de líneas: 1 línea obligatoria
├─ Máximo de líneas: No especificado (depende del backend)
├─ Cantidad: No puede ser 0 ni negativa
├─ Cantidad: No puede exceder el stock disponible del producto
├─ Cliente: No puede cambiar después de guardar (algunas versiones)
├─ Descuento: Generalmente entre 0% y 100%
├─ Estado: Por defecto es "Completada" al guardar
└─ Stock: Se actualiza automáticamente cuando se guarda la venta

                                    APÉNDICE: NOTAS IMPORTANTES
═══════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════

1. VALIDACIONES EN CLIENTE Y SERVIDOR:
   - Todas las validaciones se hacen en el frontend (HTML/JavaScript)
   - El backend (.NET Core) hace validaciones adicionales más rigurosas
   - Nunca confiar SOLO en validaciones del cliente

2. MÁSCARAS AUTOMÁTICAS:
   - Se aplican en tiempo real mientras el usuario escribe
   - No permiten caracteres no válidos
   - Los guiones/separadores se agregan automáticamente

3. BÚSQUEDA:
   - Busca en tiempo real sin necesidad de presionar Enter
   - Se filtran resultados mientras se escribe
   - Es case-insensitive (no diferencia mayúsculas)

4. PAGINACIÓN:
   - Máximo 10 registros por página
   - Se actualiza automáticamente
   - Botones: Primera, Anterior, Siguiente, Última

5. PERMISOS:
   - Validados en frontend para UX
   - Validados en backend para seguridad
   - Admin: Acceso total a todas las funciones
   - Usuario: Acceso limitado (no crear/editar/eliminar)

6. SOFT DELETE:
   - Los registros no se eliminan, se marcan como inactivos
   - Hay módulos separados para ver/restaurar eliminados
   - Solo ADMINS pueden ver eliminaciones

7. DATOS SENSIBLES:
   - Contraseñas se envían hasheadas al servidor
   - Tokens se guardan en localStorage
   - No se guardan datos sensibles sin encriptar

8. FORMATOS DE RESPUESTA:
   - Todos los errores van con status code HTTP apropiado
   - Éxitos van con status 200/201
   - Los mensajes de error son amigables para el usuario
